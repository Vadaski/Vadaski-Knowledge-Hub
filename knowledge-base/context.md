# 知识库上下文

## 目录概述
这是一个基于 BMAD Core 框架的个人知识库系统，用于存储和管理链接归档、讨论总结、文章输出和语义索引。该系统采用分层上下文索引机制，通过 context-provider 代理维护语义索引，并通过 knowledge-butler 代理提供知识管理服务。

## 文件摘要
- 当前目录为知识库的核心存储目录，包含所有知识内容的组织结构和索引文件

## 子目录
- **archives**: 用于归档历史链接和内容的目录，存储已处理并分类的链接资源。采用分类树状结构组织，便于快速查找。[链接](./archives/context.md)
- **discussions**: 用于存储讨论总结和话题记录的目录，包含基于知识库的讨论内容。[链接](./discussions/context.md)
- **articles**: 用于存放输出文章的目录，存储基于知识库辅助创作的文章内容。[链接](./articles/context.md)
- **indexes**: 用于维护语义索引的目录，包含知识库的语义索引和检索信息。[链接](./indexes/context.md)

## 模块架构
本知识库采用分层架构：
- **存储层**: 各子目录存储具体内容（链接、讨论、文章）
- **索引层**: indexes 目录维护语义索引，支持快速检索
- **代理层**: chat-mode 目录中的代理定义提供智能管理能力
  - context-provider: 负责创建和维护语义索引
  - knowledge-butler: 整合索引能力，提供知识管理服务

## 规则
- 所有内容必须使用中文书写
- 只归档相关知识，避免无关信息
- 定期更新索引以保持准确性
- 父级 context.md 只描述子目录职责和链接，不包含子目录内部文件细节
- 每个子目录维护独立的 context.md，描述该目录的具体内容和结构
- 索引更新时记录 Git 快照信息（如可用）

## AI 协作工作流
1. **接收链接**: 通过 knowledge-butler 处理链接，访问、理解、归档到 links 目录，并更新索引
2. **讨论话题**: 结合知识库内容进行讨论，记录想法和总结，归档到 discussions 目录
3. **辅助写作**: 基于知识库提供建议和内容支持，将文章保存到 articles 目录
4. **索引维护**: 使用 context-provider 的 create-context 和 update-context 命令维护语义索引
5. **隐私保护**: 只读取与当前对话相关的知识内容，避免不必要的文件加载
