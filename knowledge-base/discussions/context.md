# 讨论目录上下文

## 目录概述
此目录用于存储讨论总结和话题记录。讨论内容可以结合知识库中的链接、文章等资源，记录想法、总结观点，并归档保存。

## 文件摘要
- 当前目录为空，等待讨论内容填充
- 讨论文件建议使用有意义的文件名，如：`YYYY-MM-DD-话题名称.md`

## 子目录
- (无子目录)

## 模块架构
讨论目录采用扁平结构，所有讨论文件直接存储在此目录下。

## 规则
- 所有讨论内容必须使用中文书写
- 讨论文件应包含话题、日期、参与内容等元信息
- 讨论可以引用知识库中的链接、文章等资源
- 讨论总结应清晰、结构化，便于后续检索

## AI 协作工作流
1. 接收讨论请求：knowledge-butler 接收用户的讨论话题
2. 检索相关知识：基于知识库索引检索相关讨论和资源
3. 记录讨论：记录讨论内容和想法
4. 总结归档：生成讨论总结并保存到此目录
5. 更新索引：更新相关索引以包含新讨论
